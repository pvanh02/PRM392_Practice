1. Exercise 12.1 – Optimize List Rebuilds in Taskly

Rebuild Optimization Complete

Actions Taken

  • TaskTile Extraction: Extracted the task row elements from TaskListScreen into a standalone TaskTile widget class under lib/lab11/screens/task_tile.dart, configuring its constructor as const.
  • Fine-Grained Rebuild Listener (Selector): Configured a Selector<TaskProvider, List<Task>> widget inside the screen body, which restricts list redraw triggers to times when elements are explicitly added or removed from the list.
  • ValueKey Assignment: Added a ValueKey(task.id) parameters to the TaskTile objects, facilitating instant item-level updates within the widget list viewport.

Observations

  • Before Optimization: Toggling any task checkbox triggered setState() on the whole page, forcing redraw cycles on the entire ListView viewport, consuming $O(N)$ operations for $N$ items.
  • After Optimization: Toggling a checkbox notifies the provider, but because the list size remains constant, `Selector` prevents the ListView container from rebuilding. Only the individual TaskTile with the matching ValueKey redraws, lowering the cost to $O(1)$ operations.

3. Exercise 12.3 – App Size Analysis

Size Analysis Complete

Analysis Command

Executed command: flutter build apk --analyze-size --target-platform android-arm64

Size Findings Summary

  • Total Release APK Size: 19.6 MB
  • Top 3 Space-Consuming Components:
    1. Native Libraries (lib/arm64-v8a/): Consumed 17.0 MB (~86.7% of total APK size). This includes Flutter's rendering engine (libflutter.so) and Dart AOT compiled binary (libapp.so).
    2. Compiled Classes (classes.dex): Consumed 1.0 MB (~5.1% of total APK size) containing compiled Java/Kotlin bytecode dependencies.
    3. Resource Metadata (resources.arsc): Consumed 317 KB (~1.6% of total APK size) containing resource indexes.
Optimization Proposals:
  • ABI Splitting: Run flutter build apk --split-per-abi to output separate slim APK files for individual architectures (e.g. arm64-v8a only, which would be ~6–7 MB).
  • ProGuard/R8 Shrinking: Activate code stripping to remove unused native transit libraries.
